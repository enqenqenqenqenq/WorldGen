# WorldGen

## 🔹 Описание проекта
**WorldGen** — это воксельная песочница с процедурной генерацией мира, разработанная на базе движка Godot 4.6. Проект демонстрирует реализацию чанковой системы загрузки окружения, многоуровневую генерацию биомов и систему персистентности, использующую локальную базу данных SQLite для сохранения изменений ландшафта и позиции игрока.

## 🔹 Технический стек
* **Движок:** Godot 4.6
* **Язык программирования:** GDScript
* **База данных:** SQLite 
* **Алгоритмы генерации:** `FastNoiseLite` (2D и 3D шумы)

## 🔹 Основные архитектурные решения

### 1. Процедурная генерация и управление миром (`world.gd`)
* **Чанковая система:** Окружение разделено на участки (чанки) размером 16x16 блоков. Чанки динамически генерируются, загружаются и выгружаются вокруг позиции игрока в пределах заданного радиуса (`load_radius`), что оптимизирует использование памяти.
* **Сложная топология:** Формирование рельефа происходит за счет комбинирования нескольких слоев 2D-шума (континенты, холмы, хребты, бассейны). Генерация пещер реализована с использованием 3D-шума.
* **Кэширование данных:** Для повышения производительности вычисления биомов, температуры, высот и близости к воде сохраняются во временные кэш-словари (`base_biome_cache`, `height_cache` и др.).
* **Декорирование поверхности:** Автоматическое размещение деревьев, камней, цветов и кактусов в зависимости от сгенерированного биома (Plains, Desert, Forest, Rocky, FrozenShore и т.д.).

### 2. Подсистема персистентности (`database_manager.gd`)
* **Структура БД:** Используется файл `user://chunk_world.db` с тремя связанными таблицами: `worlds` (идентификаторы и сиды), `player_state` (координаты камеры) и `modified_blocks` (измененные воксели).
* **Асинхронное сохранение:** Процесс записи массива измененных блоков делегирован в отдельный фоновый поток (`Thread`). Это предотвращает зависания (фризы) основного потока рендеринга при фиксации большого объема данных.
* **Поддержка множественных сохранений:** Реализован функционал создания новых миров (генерация нового сида) и загрузки последнего активного мира при старте приложения.

### 3. Механика взаимодействия (`main.gd`, `player.gd`)
* **Трассировка лучей (Raycasting):** Функция `pick_block` использует алгоритм пошагового прохода (step-by-step) вдоль вектора взгляда камеры для точного определения координат сетки (`GridMap`), где находится целевой блок или пустое пространство для строительства.
* **Свободная камера (Free-fly):** Реализован контроллер наблюдателя без гравитации (noclip) с поддержкой ускорения для быстрого перемещения по сгенерированному миру.

## 🔹 Управление
* **W, A, S, D** — Перемещение в горизонтальной плоскости.
* **Space / Q** — Перемещение вверх / вниз.
* **Shift** — Ускорение движения.
* **Мышь** — Вращение камеры.
* **ЛКМ (Левая кнопка мыши)** — Разрушить блок.
* **ПКМ (Правая кнопка мыши)** — Установить блок.
* **1, 2, 3** — Выбор типа блока (Трава, Камень, Песок).
* **Esc** — Освободить / захватить курсор мыши.

## 🔹 Развёртывание
1. Откройте проект в Godot Engine (версия 4.6).
2. Убедитесь, что в проекте присутствует и активирован модуль/плагин для работы со встроенным SQLite.
3. Запустите основную сцену (`main.tscn`). База данных инициализируется автоматически при первом запуске.
